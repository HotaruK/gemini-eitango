# 英単語学習アプリ 仕様書

## 1. 目的 / 背景

英語を読んでいるときに出会う単語・熟語・成句・スラング・ミーム的な表現を、
その場でまとめて調べて保存し、時間が経っても忘れないようにクイズで定着させる
個人用の学習ツール。

- 単語は既存の辞書アプリで引けるが、成句・スラング・ミームはGoogle検索が必要で手間。
- 調べても時間が経つと忘れてしまう。
- 「調べる手間の削減」と「定着の仕組み」を1つのアプリで両立させる。

## 2. 全体構成

- **形態**: PWA(Progressive Web App)。スマホのブラウザで動作し、ホーム画面に追加可能。
- **バックエンド**: なし(純粋な静的アプリ)。辞書APIもGemini APIもブラウザから直接呼ぶ。
- **データ保存**: 端末内の IndexedDB。CSV でエクスポート/インポート可能。
- **対応環境**: iOS Safari / Android Chrome を主対象。PC ブラウザでも動作。

## 3. 意味調べモード

### 3.1 入力
- テキスト入力欄に単語・熟語・成句・スラング・ミームを入力して検索。

### 3.2 意味の取得フロー(辞書API + LLM 併用)
1. まず **dictionaryapi.dev(無料・キー不要)** に問い合わせる。
   - ヒットした場合: 英英定義・発音記号・品詞・例文・(あれば音声)を取得。
2. **Gemini(無料枠)** に問い合わせて以下を補完/生成する:
   - 英日訳(日本語の意味)
   - ニュアンス・使われる文脈
   - スラング/ミーム/成句の場合は由来や元ネタ
3. dictionaryapi.dev が **404(未収録=スラング・ミーム・成句など)** の場合:
   - Gemini のみで 英英定義・英日訳・例文・由来・文脈 をまとめて生成。

> 標準的な単語は無料辞書APIで賄い、Geminiの利用を最小化してコスト(無料枠)を節約する。

### 3.3 表示項目
- 見出し語(term)
- 種別(word / idiom / slang / meme / phrase)※LLMが推定
- 英日の意味(meaning_ja)
- 英英の定義(definition_en)
- 発音記号・音声(取得できた場合)
- 例文(examples)
- ニュアンス・由来メモ(note)

### 3.4 保存とフラグ
- 検索結果は自動で履歴に保存(永続化)。同じ語を再検索した場合は既存レコードを更新。
- 各語に「知らない(★)」フラグを付けられる → クイズの出題対象になる。
- フラグは後から一覧画面でも付け外し可能。

## 4. クイズモード

### 4.1 出題
- ★フラグの付いた語から四択問題を出題する。
- 問題形式(デフォルト): 「この語の意味は?」→ 英日の意味を4択で選ぶ。
- 選択肢: 正解1つ + ダミー3つ(他の保存済み語の meaning_ja からランダム抽出)。
- ダミーは種別が近い語を優先し、なければ全体から補完。

### 4.2 成績記録
各語について以下を記録:
- `quiz_count`(出題回数)
- `correct_count`(正答数)
- `correct_rate = correct_count / quiz_count`(正解率、未出題は未定義)

### 4.3 出題優先度アルゴリズム
重み付きランダム抽選で、正解率が低い語ほど出やすくする。

```
weight(word):
  未出題(quiz_count = 0)     → 最優先(固定の高い重み、例: 3.0)
  出題済み                    → weight = (1 - correct_rate) + ε
                                (ε は 0 除算・完全正解の救済用の小さな値、例: 0.1)
```
- 出題回数が少ない語にわずかな加点(探索ボーナス)を加えてもよい。

### 4.4 習得判定
- `quiz_count >= 5` かつ `correct_rate >= 0.8` を満たしたら「習得済み」とみなす。
- 習得済みの語は出題頻度を下げる(重みを大幅に減らす)。★フラグの自動解除はしない
  (誤操作防止のため手動解除のみ。設定で自動解除に変更可)。

### 4.5 エッジケース
- 出題可能な★語が4未満の場合: ダミーを保存済み全体から補完。それでも4に満たなければ
  「クイズには最低4語必要」と案内する。

## 5. データモデル

```
Word {
  id            : string  // term を正規化したキー or UUID
  term          : string  // 見出し語(単語・熟語・成句)
  type          : enum    // word | idiom | slang | meme | phrase
  meaning_ja    : string  // 英日
  definition_en : string  // 英英
  phonetic      : string? // 発音記号
  audio_url     : string? // 音声(あれば)
  examples      : string[]
  note          : string  // ニュアンス・由来
  source        : enum    // dictionary | gemini | mixed
  is_flagged    : boolean // クイズ対象か
  quiz_count    : number
  correct_count : number
  created_at    : number  // epoch ms
  updated_at    : number
}
```

## 6. データのエクスポート/インポート

- **エクスポート**: 全 Word レコードを CSV(UTF-8, BOM付き)で書き出しダウンロード。
  - 例文・改行を含むため、RFC4180 準拠のクォート処理を行う。
- **インポート**: CSV を読み込んで復元。`id`/`term` が一致するレコードはマージ(更新)。
- バックアップ用途を主とする。将来 JSON エクスポートも追加可能。

## 7. 設定

- Gemini API キーの入力・保存(端末内 localStorage)。
  - キーはユーザー自身が Google AI Studio で取得したものを入力。
  - 個人利用前提でクライアント保存する旨を注記。
- 出力言語(英日訳の詳しさ)などの軽微な設定は将来拡張。

## 8. 画面構成(案)

- **タブ1: 調べる** — 検索欄 + 結果カード + ★フラグボタン + 保存
- **タブ2: 単語帳** — 保存済み一覧(検索・種別/フラグでフィルタ、成績表示、編集/削除)
- **タブ3: クイズ** — 出題・回答・即時フィードバック・セッション成績
- **設定** — APIキー、エクスポート/インポート

## 9. 技術スタック(推奨・要確認)

- ビルド: Vite + TypeScript
- UI: React(または軽量に Preact)
- 保存: IndexedDB(Dexie ラッパー)
- PWA: manifest.json + Service Worker(オフラインでも単語帳/クイズは動作。検索はオンライン必須)
- ホスティング: 静的ホスティング(GitHub Pages / Netlify)またはローカル

## 10. スコープ外(現時点)

- ユーザー認証・クラウド同期(端末内のみ)
- SNS 共有・複数ユーザー
- 発音判定・スペル入力クイズ(四択のみ)
